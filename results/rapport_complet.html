            2. Architecture du Réseau


            2.1 Topologie


            Le réseau est une architecture dense à 2 couches:


            
Entrée (d neurones)
    ↓
Couche cachée (h neurones, ReLU)
    ↓
Couche sortie (C neurones, Softmax)
    ↓
Sortie (probabilités)

            Paramètres:

            • d = 64 (B&W 8×8) ou 768 (couleur 16×16×3)
              • h ∈ {4, 8, 32, 64} (varie par étape)
                • C = 3 (étapes A-C) ou 5 (étape D)

                  • 2.2 Équations Mathématiques


                  Forward Pass:

                  • z₁ = W₁x + b₁
                    • a₁ = ReLU(z₁)
                      • z₂ = W₂a₁ + b₂
                        • ŷ = softmax(z₂)

                          • Loss (Cross-Entropy):

                          • L = -Σ yᵢ log(ŷᵢ)

                            • Backward Pass:

                            • δ₂ = ŷ - y
                              • δ₁ = (W₂ᵀδ₂) ⊙ ReLU'(z₁)
                                • ∂L/∂W = δ ⊗ aᵀ

                                  • Update Rule (SGD):

                                  • θ ← θ - α∇L

                                    • 2.3 Hyperparamètres


                                    | Paramètre | Valeur | Notes | |-----------|--------|-------| | Learning Rate | 0.1 (0.05 XL) | SGD simple | | Activation Cachée | ReLU | f(z) = max(0, z) | | Activation Sortie | Softmax | Multi-class | | Loss Function | Cross-Entropy | Classification | | Optimiseur | SGD | Pas de momentum | | Epochs | 50-100 | Varie par étape | | Batch Size | 1 | Online learning | | Initialisation | Xavier Uniform | Distribution adaptée |

                                                    5. Résultats Étape D: Couleur + 5 Classes


                                                    5.1 Données


                                                    Images B&W: 8×8 grayscale, 64 pixels

                                                    Images Couleur: 16×16 RGB, 768 pixels (12× plus grands)

                                                    Classes étendues: Happy, Neutral, Sad, Angry, Surprised (5 classes)

                                                    Complexité: 5 classes vs 3, pixels + couleurs = beaucoup plus difficile


                                                    5.2 Comparaison B&W vs Couleur


                                                    | Aspect | B&W 3-class | Couleur 5-class | |--------|------------|-----------------| | Entrée | 64 pixels | 768 pixels | | Classes | 3 | 5 | | Test Accuracy | 100% | 20% | | Complexité | Basse | Haute |

                                                    5.3 Analyse


                                                    📊 Observation: Performance réduite sur couleur (20% vs 100% B&W)


                                                    🔍 Raisons:

                                                    • Dataset beaucoup plus complexe (5 classes vs 3)
                                                      • Dimension d'entrée 12× plus grande
                                                        • Probablement besoin de plus d'epochs/données
                                                          • Architecture trop simple pour cette complexité

                                                            • ✅ Positive: Démontre l'extensibilité du système

                                                            6. Détails d'Implémentation


                                                            6.1 Composants Clés


                                                            • **model.py:** Classe NeuralNetwork avec forward, backward, loss, predict
                                                              • **generate_data.py:** Création d'images PGM synthétiques (smileys)
                                                                • **generate_data_color.py:** Création d'images PPM couleur
                                                                  • **image_loader.py:** Parsing manuel PGM/PPM, normalisation
                                                                    • **validation.py:** Stratified split train/val/test
                                                                      • **metrics.py:** Confusion matrix, accuracy, precision, recall, F1

                                                                        • 6.2 Stabilité Numérique


                                                                        ✅ Softmax: Soustraction du max(z) avant exp() pour éviter overflow

                                                                        ✅ Cross-entropy: Clipping des probabilités [1e-10, 1-1e-10] pour éviter log(0)

                                                                        ✅ Initialisation: Xavier uniform distribution

                                                                        ✅ Gradient: Normalisation par batch size
